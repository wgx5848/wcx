1. HTML5 语义化页面布局技术
使用 header、main、section 等 HTML5 语义标签划分页面结构，分别搭建页面标题区、搜索交互区、提示信息区、天气展示卡片、三日预报模块，相比普通 div 布局结构更清晰，便于代码阅读与维护。页面中通过 id 属性绑定输入框、按钮、数据展示容器，为后续 JS 获取 DOM 元素提供唯一标识。
2. CSS3 页面样式与响应式布局技术
线性渐变背景：采用 linear-gradient 实现蓝调渐变页面背景，美化页面视觉效果；
Flex 弹性布局：用于搜索栏横向排布、实时温度信息左右排版，自动适配容器宽度；
Grid 网格布局：将天气详情分为两栏、三日预报分为三栏，规整卡片排版；
媒体查询响应式适配：通过 @media 判断屏幕宽度，手机端自动将多栏网格改为单列，实现电脑、手机两端兼容；
基础美化样式：设置圆角、阴影、内边距、鼠标悬浮变色，提升页面交互观感。
3. JavaScript DOM 元素操作技术
通过 getElementById 获取页面所有交互与展示 DOM 节点，封装统一的显示、隐藏方法控制加载提示、错误提示、天气卡片的切换显示；使用 createElement 动态创建预报卡片 DOM，innerHTML 拼接天气文字内容，实现数据动态渲染，无需手写固定页面结构。
4. ES6 JavaScript 新语法技术
箭头函数：简化函数定义写法，代码更简洁；
async/await 异步语法：搭配 setTimeout 模拟网络请求延迟，展示加载动画，模拟真实接口请求等待效果；
对象解构：快速提取天气对象中的城市、温度、风力等字段，简化数据读取代码；
模板字符串：使用${变量}快速拼接文字与数据，拼接预报内容更便捷。
5. localStorage 本地持久化存储技术
利用浏览器内置 localStorage 存储用户上一次查询的城市名称，页面刷新、关闭重开后自动读取缓存数据，无需重复输入城市，实现查询记录持久保存，提升使用体验。
6. 本地 JSON 模拟数据源技术
采用数组对象 JSON 格式存储湛江、霞山、禅城、广州等广东市区的实时天气、三日预报模拟数据，替代第三方天气 API。无需注册接口、无需密钥、不存在跨域报错，断网环境下也能正常查询天气，作为项目兜底数据方案。
7. 交互事件监听技术
按钮点击事件：监听查询按钮点击，触发城市天气查询逻辑；
键盘回车事件：监听输入框回车按键，支持回车快速查询，优化操作便捷性；
定位按钮点击事件：绑定定位按钮触发地理定位逻辑。
8. 浏览器 Geolocation 地理定位 API
调用浏览器原生定位接口获取用户当前位置，若浏览器不支持定位或用户拒绝授权，自动兜底加载湛江天气数据，兼容各类浏览器权限场景。
整体业务实现流程
页面加载完成后读取本地存储的上次查询城市并自动加载天气；用户输入城市后点击按钮或按回车，程序校验输入内容，开启加载动画，匹配本地模拟天气数据；匹配成功后渲染实时天气与三日预报并保存当前城市，匹配失败则弹出错误提示；点击定位按钮尝试获取用户地理位置，异常时自动展示湛江天气。
